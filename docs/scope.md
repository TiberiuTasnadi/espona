# Vision Document

## Nombre del proyecto

RuralOps

## Resumen

RuralOps es una aplicación web diseńada para ayudar a pequeńas explotaciones agrícolas y cooperativas a registrar y consultar la actividad realizada en sus parcelas.

Actualmente gran parte de esta información se gestiona mediante hojas de cálculo, grupos de mensajería o incluso papel. Esto provoca pérdida de información, dificultad para consultar históricos y poca visibilidad sobre el estado real de la explotación.

El objetivo de RuralOps es ofrecer una herramienta sencilla que permita registrar y consultar la información esencial del día a día agrícola.

## Problema

Los agricultores y pequeńas cooperativas necesitan registrar información relacionada con:

* Parcelas
* Cultivos
* Trabajos realizados
* Históricos de actividad

Las soluciones existentes suelen ser excesivamente complejas, costosas o incluyen funcionalidades innecesarias para explotaciones pequeńas.

## Objetivo

Crear una aplicación sencilla y fácil de utilizar que permita mantener un registro digital de la actividad agrícola.

## Público objetivo

### Agricultor

Responsable de una o varias parcelas.

Necesita registrar los trabajos realizados y consultar información histórica.

### Responsable de explotación

Persona encargada de supervisar varias parcelas y coordinar trabajos.

Necesita conocer el estado general de la explotación.

### Cooperativa

Organización que presta servicios a varios agricultores.

Necesita disponer de información centralizada y accesible.

## Propuesta de valor

* Simplicidad
* Facilidad de uso
* Históricos centralizados
* Información accesible desde cualquier lugar
* Coste reducido

## Objetivos del MVP

* Registrar parcelas
* Registrar cultivos
* Registrar trabajos realizados
* Consultar históricos
* Gestionar usuarios básicos

## Criterios de éxito

* Un agricultor puede registrar una actividad en menos de un minuto.
* La información histórica puede consultarse fácilmente.
* El sistema permite conocer el estado actual de las parcelas.

## Fuera del alcance

Las siguientes funcionalidades no forman parte del MVP:

* Facturación
* Gestión económica
* Gestión de almacén
* Compras
* Ventas
* Contabilidad
* Inteligencia artificial
* Integraciones externas

Estas funcionalidades podrán evaluarse en futuras versiones.
